# iOS 历史详情 Markdown 与段落卡片增强设计

**日期：** 2026-04-06  
**范围：** `iTime-iOS/UI/Conversation/iOSConversationView.swift` 中历史详情页（复盘详情）所有 Markdown 文本区块

## 背景与目标

当前 iOS 复盘历史详情页在展示层存在三个问题：

1. Markdown 高亮语法与代码块语法在部分区块未统一渲染
2. “客观总结”段落视觉层级与正文接近，缺少次级信息感
3. 段落容器存在透明感过强的问题，不符合流程图区块底卡的半透明视觉基线

目标是在不改数据模型和业务流程的前提下，统一历史详情页 Markdown 渲染行为，并使段落视觉与已有流程图卡片风格对齐。

## 设计方案

### 1. 统一 Markdown 渲染入口

在历史详情页视图内部建立统一渲染函数，供以下区块复用：

- 总结（含“客观总结”“主观评价”）
- 发现
- 建议
- 流水账复盘正文

该入口负责文本归一化与样式应用，避免各区块独立渲染导致行为不一致。

### 2. Markdown 与代码块样式

统一启用：

- 行内代码：等宽字体 + 轻量背景
- 代码块：可横向滚动容器 + 半透明背景卡片
- 强调文本：保持高可读性强调（粗体/高亮）

并保留对已有兼容输入的归一化处理：

- `\\n` 转换为换行
- `==highlight==` 与 `<mark>...</mark>` 转换为可显示的强调格式

### 3. 客观总结小字号

“客观总结”使用紧凑渲染模式（compact）：

- 前景色更弱（次要文本）
- 字号相较正文下调（例如 `0.9em`）

“主观评价”维持正文层级，确保信息层次清晰。

### 4. 段落半透明卡片

对段落块（paragraph）统一加底卡，视觉参数与流程图区域底卡一致：

- 填充：`Color.secondary.opacity(0.08)`
- 边框：`Color.secondary.opacity(0.15)`，1pt
- 圆角：连续圆角（14）

这样每个段落都具备可见承载面，不再出现“全透明段落”问题。

## 影响面与兼容性

- 仅影响 iOS 历史详情页读态展示
- 不改编辑态输入与保存逻辑
- 不改 AI 生成、存储、分享、归档数据结构
- 与现有主题体系兼容（包括流光与纯净主题）

## 错误处理与边界

- 空内容保持原有空态逻辑
- 非法/不规范 Markdown 由 MarkdownUI 按普通文本降级渲染
- 代码块过长通过横向滚动避免截断

## 验证要点

1. 历史详情页四类文本区块都可正确渲染 Markdown
2. 行内代码与 fenced code block 均可见且可读
3. “客观总结”字号与正文形成明显层级
4. 每个段落都显示半透明卡片，不再全透明
