# iOS 自定义主题方块容器重做设计

## 背景与目标

当前 iOS 设置页的自定义主题网格存在三个问题：

1. 主题图片内容会出现超出圆角方块容器的视觉溢出，容器约束不够严格。
2. 长按菜单触发位置与用户操作的具体主题方块不一致，交互归属不清晰。
3. 自定义主题仍依赖名称字段（`displayName`），与当前“仅视觉预设”定位不一致。

本次改造目标是将主题预设统一为“图片裁切预设卡片”，保证容器内展示一致、交互命中准确，并移除名称属性。

## 范围

### In Scope

- iOS 设置页“主题选择网格”的自定义主题方块样式与交互改造。
- `CustomThemePreset` 数据模型移除 `displayName`。
- 本地持久化兼容旧数据（旧数据中可能含 `displayName`）。
- 相关单元测试/回归测试更新。

### Out of Scope

- 编辑页顶部预览卡片样式与尺寸规则。
- 新增主题标签、分组、排序规则等功能。
- macOS 端 UI 调整。

## 方案概述（采用方案 B）

引入统一的“主题方块容器”组件承接所有自定义主题项，容器负责：

- 固定方形尺寸与圆角裁切；
- 图片铺满并裁切（不留白）；
- 命中区域与视觉区域一致；
- 点击应用主题；
- 长按弹出该容器专属菜单（仅编辑/删除）。

同时在数据层移除主题名称字段，主题项只保留与视觉结果相关的数据（图片与裁切参数）。

## 详细设计

### 1. 组件与层级

- 新增 `ThemePresetContainer`（命名可按代码风格微调），用于包装每个主题方块内容。
- `ThemePresetTile` 改为通过该容器渲染内容，禁止在容器外叠加会突破圆角边界的层。
- `CustomThemePresetCard` 去除底部文本层，卡片内容仅图片。

### 容器行为

- 尺寸：继续遵循当前网格计算得到的边长（不改变布局算法）。
- 裁切：圆角路径一次性约束背景、图片、覆盖层和命中区域。
- 图片缩放：使用“铺满并裁切”策略，保证无留白。
- 交互：
  - 点击容器：立即应用该主题；
  - 长按容器：弹出该主题的上下文菜单，菜单项为“编辑 / 删除”。

### 2. 数据模型与持久化

- `CustomThemePreset` 删除 `displayName` 字段。
- 更新相关初始化调用与保存 API：
  - `saveCustomThemePreset(...)` 移除 `displayName` 参数；
  - 编辑逻辑和新增逻辑仅处理图片与裁切参数。

### 旧数据兼容策略

- 保留 `Codable` 兼容读取能力：当本地旧 JSON 包含 `displayName` 时，解码应成功并忽略该字段。
- 重新保存后按新结构写回，不再写入名称字段。
- 兼容目标：用户已保存主题不丢失、不重置。

### 3. 数据流与交互流

1. 用户点击某个主题方块。
2. 容器触发应用动作，更新 `selectedCustomThemePresetID` 与当前主题参数。
3. 用户长按同一方块。
4. 该容器弹出菜单，执行编辑或删除对应 preset。
5. 删除当前选中主题时，沿用既有回退规则（切到下一个或恢复默认）。

### 4. 错误处理与边界

- 找不到 preset ID 时沿用当前安全行为（不执行更新）。
- 旧数据解码失败时沿用现有默认回退路径，不引入静默“伪成功”状态。
- 保持裁切参数的现有 clamp 规则，避免异常值导致视觉错位。

### 5. 测试策略

- 更新 `UserPreferencesTests` 中所有 `saveCustomThemePreset` 调用，移除名称参数依赖。
- 新增兼容性测试：构造包含 `displayName` 的旧格式 `customThemePresets` 数据，验证仍可解码并正确读取图片与裁切参数。
- 若存在设置页交互测试，补充断言：
  - 点击方块触发应用；
  - 长按方块出现“编辑/删除”；
  - 方块不渲染名称文本层。

## 验收标准

1. 主题网格中每个自定义主题都只显示在圆角方形容器内，内容不越界。
2. 点击某方块立即应用该主题。
3. 长按某方块时，菜单由该方块触发且仅有“编辑/删除”。
4. 代码层不再暴露 `CustomThemePreset.displayName`。
5. 旧用户已保存主题可无损读取并继续使用。
