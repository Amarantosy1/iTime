# iTime 中文化与设置窗口设计

## 目标

在不改动主工作区的前提下，于独立 `worktree` 中完成三项改动：
- 将当前应用中的用户可见英文文案改为中文
- 增加原生 macOS 设置窗口，并将日历选择功能放入设置中
- 保留手动勾选/取消勾选日历的能力，不增加“节假日”自动过滤逻辑

## 当前现状

- 菜单栏主界面和详情窗口中仍有英文文案
- `AppModel` 已经具备日历列表读取、选中状态计算与持久化能力
- `UserPreferences.selectedCalendarIDs` 已经能保存用户选择
- 当前没有设置窗口，日历选择 UI 也还未独立出来

## 设计决策

### 1. 设置入口

菜单栏主界面增加“设置”按钮，点击后打开原生 macOS `Settings` 窗口。

原因：
- 符合用户要求的“原生设置窗口”
- 将统计入口与配置入口分离，结构更清晰
- 后续如果增加更多偏好项，可以继续沿用同一入口

### 2. 设置页内容

新增设置视图，展示当前可访问的日历列表，并为每个日历提供勾选开关。

行为规则：
- 首次授权后，如果用户尚未保存任何选择，则默认选中全部日历
- 后续修改直接写回 `selectedCalendarIDs`
- 修改后立即刷新统计结果，菜单栏与详情窗口共享同一份 `AppModel`

### 3. 文案中文化

以下内容统一改为中文：
- 时间范围标题
- 菜单栏主界面文案
- 详情窗口标题与空状态文案
- 授权提示与按钮文案
- 设置窗口标题与日历选择说明

### 4. 日历过滤策略

不新增“节假日”自动过滤逻辑。

用户如果不希望某个订阅日历参与统计，应在设置窗口中手动取消勾选。这样能保持逻辑简单，也避免对不同语言或不同来源的节假日日历做不稳定判断。

## 影响范围

- `AppModel`：补充设置视图所需的选择接口与刷新行为
- `iTimeApp`：注册原生设置窗口
- `MenuBarContentView`：增加设置按钮并移除任何未来应放入设置的选择入口
- 新增设置视图：承载日历选择 UI
- `TimeRangePreset` 与多个现有视图：完成中文文案替换
- 测试：增加中文标题和选择逻辑的覆盖
