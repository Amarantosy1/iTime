# iTime Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the app reliably follow macOS light and dark appearance, with the overview window using a dark-appropriate background and no manual theme toggle.

**Architecture:** Add one lightweight theme helper in `UI/Theme` that exposes a small, testable background palette model and a SwiftUI gradient builder. Keep system materials for cards, and consume the helper from the overview window while only making small semantic-color adjustments elsewhere if needed.

**Tech Stack:** SwiftUI, Swift Testing, Xcode project + SwiftPM

```swift
import SwiftUI

enum AppTheme {
    struct BackgroundPalette: Equatable {
        let startHex: String
        let endHex: String
    }

    static func overviewBackgroundPalette(for colorScheme: ColorScheme) -> BackgroundPalette {
        switch colorScheme {
        case .dark:
            BackgroundPalette(startHex: "#111827", endHex: "#1F2937")
        default:
            BackgroundPalette(startHex: "#F2F7FF", endHex: "#E6F0FA")
        }
    }
}
```

```swift
enum AppTheme {
    struct BackgroundGradient {
        let palette: BackgroundPalette

        var linearGradient: LinearGradient {
            LinearGradient(
                colors: [Color(hex: palette.startHex), Color(hex: palette.endHex)],
                startPoint: .topLeading,
                endPoint: .bottomTrailing
            )
        }
    }

    static func overviewBackgroundGradient(for colorScheme: ColorScheme) -> BackgroundGradient {
        BackgroundGradient(palette: overviewBackgroundPalette(for: colorScheme))
    }
}
```
